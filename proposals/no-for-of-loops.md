## Validation prompt

FP-aware guidance for the [react-review agent](https://github.com/millionco/react-review) when triaging this rule:

> Flag this only when the `for...of` is in shipped code that matters for bundle size. Typical false positives are `for await...of` async iterators, loops that genuinely need `break`/`continue` or early `return`, and server-only or build-time scripts where the iterator cost does not affect the client bundle. Also be careful not to treat intentional imperative control flow as a mistake just because a loop could be rewritten.

## Fix prompt

Actionable fix suggestion surfaced to the user when the rule fires:

> Replace the `for...of` with `.forEach()` or another non-iterator pattern when you only need to visit every item. For example:

```ts
const changedKeys: string[] = [];
keys.forEach((key) => {
  if (prev[key] !== next[key]) changedKeys.push(key);
});
```

Keep `for...of` only when you truly need early exit or async iteration.
