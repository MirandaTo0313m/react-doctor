## Validation prompt

FP-aware guidance for the [react-review agent](https://github.com/millionco/react-review) when triaging this rule:

> Check whether the logical expression is actually a standalone statement. Legitimate false positives are rare, but this should not be applied to JSX conditional rendering (`{condition && <Foo />}`), assignments, returns, or other expressions where the value is used. Also watch for generated or test-only code that intentionally uses terse guard syntax, though an explicit `if` is usually clearer there too.

## Fix prompt

Actionable fix suggestion surfaced to the user when the rule fires:

> Replace the short-circuit statement with an explicit conditional. For example:

```tsx
if (enabled) {
  onRun();
}
```

If you were only trying to compute a value, move the expression into an assignment or return instead of leaving it as a bare statement.

## Proposed AST detector

Would land at `packages/oxlint-plugin-react-doctor/src/plugin/rules/correctness/no-short-circuit-statements.ts`:

```ts
import { defineRule } from "../../utils/define-rule.js";
import { isNodeOfType } from "../../utils/is-node-of-type.js";
import type { EsTreeNodeOfType } from "../../utils/es-tree-node-of-type.js";
import type { Rule } from "../../utils/rule.js";
import type { RuleContext } from "../../utils/rule-context.js";

const SHORT_CIRCUIT_OPERATORS = new Set(["&&", "||"]);

export const noShortCircuitStatements = defineRule<Rule>({
  id: "no-short-circuit-statements",
  severity: "warn",
  recommendation:
    "Use an explicit `if` block instead of a standalone short-circuit expression statement",
  create: (context: RuleContext) => ({
    ExpressionStatement(node: EsTreeNodeOfType<"ExpressionStatement">) {
      if (!isNodeOfType(node.expression, "LogicalExpression")) return;
      if (!SHORT_CIRCUIT_OPERATORS.has(node.expression.operator)) return;

      context.report({
        node,
        message:
          "Standalone short-circuit expression statement — use an explicit `if` block instead",
      });
    },
  }),
});
```
